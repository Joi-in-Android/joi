## Zohar 指令：Android APK 构建→安装→启动 完整协议（2026-04-05）

### 核心认知纠正

**我运行在：** Termux → root → chroot Ubuntu 24.04
**我 ≠ Android 宿主本身**

- 构建：在 chroot 内完成
- 文件导出：通过共享路径 / bind mount / cp
- 安装 APK：**必须调用宿主侧的 pm / cmd package**
- 启动应用：**必须调用宿主 Activity Manager**

**禁止表述：** "容器内直接安装 app"、"在 chroot 里安装 APK"
**正确表述：** 在 chroot 内构建，在宿主侧安装和启动

### 完整执行步骤（不可乱序）

1. **识别项目** — package name、模块名、APK 输出路径、主 Activity
2. **chroot 内构建** — `./gradlew :app:assembleDebug`
3. **验证 APK 真实存在** — `find` + `ls -lh` + `sha256sum`
4. **导出到宿主路径** — `/data/local/tmp/` 或 `/sdcard/Download/`
5. **宿主侧安装** — `su -c 'pm install -r /path/to.apk'`
6. **查询包版本** — `pm path <pkg>` + `dumpsys package <pkg>`
7. **启动应用** — `am start -n <pkg>/.MainActivity` 或 `monkey -p <pkg> -c android.intent.category.LAUNCHER 1`
8. **记录证据** — 路径、SHA256、安装输出、包信息、启动结果
9. **失败时停在该步，保留原始输出，给出具体原因**

### 成功判定标准（必须同时满足）

- APK 已构建完成（文件真实存在，非零字节）
- APK 已导出到宿主路径
- `pm install` 输出包含 **Success**
- `pm path <pkg>` 能查到目标包
- `dumpsys package <pkg> | grep version` 符合预期
- `am start` 或 `monkey` 启动成功

### 禁止行为

- 没检查 APK 文件就说"已编译"
- 没检查安装输出就说"已安装"
- 安装失败继续说"已部署"
- 启动失败却说"运行正常"
- **编造版本号、编造启动结果**
- 把 chroot 当成 Android 宿主

### 交付格式（结构化 JSON）

成功：
```json
{
  "ok": true,
  "apkPath": "/data/local/tmp/xxx.apk",
  "apkSha256": "...",
  "packageName": "com.xxx",
  "installOutput": "Success",
  "installed": true,
  "versionName": "x.x.x",
  "launchOk": true
}
```

失败：
```json
{
  "ok": false,
  "stage": "build|export|install|verify|launch",
  "error": "具体错误",
  "evidence": "原始输出"
}
```

1. 接收 APK 路径

5. 启动应用并输出标准化结果
